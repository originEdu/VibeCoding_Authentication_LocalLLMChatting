NPC 대화 기능 설계

사용자가 자유롭게 입력한 질문에 NPC가 답변하는, UE5 + 로컬 LLM 대화 기능

NPC의 두뇌 = http://localhost:8080 (llama.cpp + Gemma-4-E2B, OpenAI 호환 API)

UE5 C++ FHttpModule 블로킹 응답 멀티턴 로그인 게이팅 프로필 오버레이
작성일: 2026-07-16 · 최종 갱신: 2026-07-16 (구현 완료)

1 배경 & 요구사항

언리얼 프로젝트(ue5/ViveCodingUE)에 정해진 문답이 아닌 자유 입력형 NPC 대화 기능을 추가한다. 이 프로젝트에는 이미 AuthClient 플러그인이 있어 FHttpModule 기반 비동기 HTTP + JSON 파싱 + UMG 위젯 + 델리게이트 콜백 패턴이 완성되어 있다. 새 기능은 이 검증된 패턴을 거의 그대로 복제해 위험을 낮춘다.

항목결정이유
응답 수신블로킹 (완성 응답 한 번에)기존 코드가 이미 이 방식 → 스트리밍 대비 구현 단순
대화 기억멀티턴 (메시지 배열 누적)이전 질문/답변을 반영한 자연스러운 대화
NPC 정체성지금은 없음나중에 한 줄로 채울 수 있게 빈 SystemPrompt 필드만 준비
로그인 게이팅로그인 성공 후에만 채팅 진입채팅이 로그인 후의 메인 화면
프로필 열기오버레이 (SwapTo 아님)채팅 위에 얹었다 닫으면 대화가 그대로 유지됨
배치기존 AuthClient 플러그인채팅 위젯이 UAuthScreenWidget 공유 헬퍼를 상속 → 동일 모듈이 가장 단순

2 화면 흐름 (구현 결과)

WBP_Login →로그인 성공 WBP_Chat (허브) →/v1/chat/completions 로컬 LLM
WBP_Chat →프로필보기 (오버레이) WBP_Profile →돌아가기 = 닫기 WBP_Chat (그대로)
  • 로그인 성공(및 저장된 토큰으로 자동 로그인) 시 곧바로 채팅 화면으로 이동.
  • 프로필보기는 SwapTo로 교체하지 않고 채팅 위에 프로필을 AddToViewport로 얹는다 → 채팅 인스턴스와 대화 로그가 살아있음.
  • 돌아가기는 프로필이 RemoveFromParent 하여 아래 채팅을 그대로 다시 드러냄.
  • 로그아웃은 프로필 오버레이를 닫고, 아래 채팅이 대화 이력을 비운 뒤 로그인 화면으로 복귀.

3 구현 (파일별)

1) UChatSubsystem — AuthSubsystem 미러링

파일: Public/ChatSubsystem.h, Private/ChatSubsystem.cpp

  • UGameInstanceSubsystem 상속, AUTHCLIENT_API
  • 상태: BaseUrl(기본 http://localhost:8080), SystemPrompt(빈 문자열), TArray<FChatMessage> History ({ Role, Content })
  • SendMessage(UserText) — user 메시지 추가 → /v1/chat/completions 바디 조립(SystemPrompt 있으면 맨 앞 system) → 전송 + HandleChat
  • ClearHistory(), SetBaseUrl(), SetSystemPrompt()
  • HandleChat — choices[0].message.content 파싱 → assistant 이력 추가 → OnChatResponse.Broadcast(...)
  • 델리게이트: FChatResult(bool bSuccess, const FString& Reply) → UPROPERTY(BlueprintAssignable) OnChatResponse
{ "messages": [{"role":"user","content":"안녕?"}], "stream": false, "temperature": 0.7 }

2) UAuthChatWidget — 로그인 후 메인 허브

파일: Public/AuthChatWidget.h, Private/AuthChatWidget.cpp

  • UAuthScreenWidget 상속
  • BindWidget: InputBox, SendButton, ChatLog, StatusText / BindWidgetOptional: ProfileButton
  • 네비게이션: ProfileWidgetClass(WBP_Profile), LoginWidgetClass(WBP_Login — 로그아웃 복귀)
  • NativeConstruct: 버튼 바인딩 + OnChatResponse 구독 + OnLogoutCompleted 구독
  • OnProfileClicked: CreateWidget(ProfileWidgetClass) + AddToViewport(10) — 채팅은 그대로 두고 위에 얹음
  • OnLogoutResult: ClearHistory() 후 SwapTo(LoginWidgetClass)

3) UAuthProfileWidget — 채팅 위 오버레이

파일: Public/AuthProfileWidget.h, Private/AuthProfileWidget.cpp

  • InfoText, LogoutButton(기존) + BackButton(BindWidgetOptional)
  • 화면 전환 대상 클래스 불필요 — 오버레이라 닫기만 하면 됨
  • OnBackClicked: RemoveFromParent() (닫기 → 아래 채팅 노출)
  • OnLogoutResult: RemoveFromParent() (오버레이만 닫음, 로그인 복귀는 채팅이 담당)
  • 생성 시 GetMe()로 내 정보 표시(기존 동작 유지)

4) UAuthLoginWidget — 성공 시 채팅으로

  • 로그인 성공(및 자동 로그인) 시 이동 대상을 ProfileWidgetClass → ChatWidgetClass(WBP_Chat)로 변경

4 에디터 배선 (코드로 불가)

위젯 BP부모 클래스배치할 위젯(이름 일치)클래스 지정
WBP_LoginAuthLoginWidget(기존)Chat Widget Class = WBP_Chat
WBP_ChatAuthChatWidgetInputBox, SendButton, ChatLog, StatusText, ProfileButtonProfile Widget Class = WBP_Profile, Login Widget Class = WBP_Login
WBP_ProfileAuthProfileWidgetInfoText, LogoutButton, BackButton(없음)

주의 프로필 위젯은 화면을 꽉 채우는 불투명 배경을 둔다. 안 그러면 아래 채팅이 비쳐 보이고, 가려지지 않은 영역의 채팅 버튼이 클릭될 수 있다.

5 검증

  • 완료 컴파일: ViveCodingUEEditor 빌드 성공 — ChatSubsystem, AuthChatWidget, AuthProfileWidget, AuthLoginWidget 컴파일·링크(에러/경고 0)
  • 플레이 테스트 (localhost:8080 실행 + 에디터 배선 후):
    • 로그인 성공 → 채팅 진입
    • "안녕?" → NPC 답변 표시
    • "방금 뭐라고 했어?" → 직전 맥락 반영(멀티턴)
    • 프로필보기 → 채팅 위에 뜸 → 돌아가기 → 대화 로그 그대로 유지
    • 로그아웃 → 로그인 복귀, 재로그인 시 이전 대화 미혼입
    • 서버 콘솔에 매 전송마다 /v1/chat/completions 요청 확인

Auth Server + UE5 Client

FastAPI 기반 JWT 인증 백엔드와, 이를 호출하는 Unreal Engine 5 클라이언트 플러그인

FastAPI JWT MySQL SQLAlchemy 2.0 UE 5.7 · C++

작성일 2026-07-15 · 최신 갱신: UI 3화면 분리 + StatusText UX

1 개요

언리얼 엔진 5에서 사용할 인증 시스템을 서버 + 클라이언트 두 파트로 구성했다. 두 파트는 하나의 REST API 계약(요청/응답 형식)을 공유한다.

파트위치상태
FastAPI 인증 서버server/테스트 12개 통과 + 실 MySQL 검증
UE5 클라이언트 플러그인ue5/AuthClient/ · Plugins/UE 5.7 빌드 + PIE · 3화면 분리
UE5 테스트 프로젝트ue5/ViveCodingUE/WBP_Login/Signup/Profile + GameMode

2 핵심 결정 사항

  • 언어/프레임워크: Python + FastAPI
  • DB: MySQL (PyMySQL 동기), SQLAlchemy 2.0 ORM + Alembic 마이그레이션
  • 인증: JWT Access + Refresh 토큰
    • Access: 30분, 무상태 검증
    • Refresh: 14일, DB에 SHA-256 해시로 저장 → 로그아웃·탈퇴 시 무효화
  • 탈퇴: 소프트 삭제 (is_active=false + 모든 refresh 토큰 무효화)
  • UE5: C++ HTTP 서비스(Subsystem) + 토큰 저장 + UMG 위젯
  • async 스택은 학습 복잡도 때문에 의도적으로 배제

3 서버 server/

구조 (계층 분리)

app/
  main.py              # FastAPI 앱, 라우터 등록
  core/
    config.py          # .env 설정 로드
    database.py        # SQLAlchemy engine/session, get_db
    security.py        # 비밀번호 해싱(bcrypt), JWT 발급/검증, 토큰 해시
  models/              # User, RefreshToken ORM
  schemas/auth.py      # Pydantic 요청/응답
  services/auth_service.py  # 비즈니스 로직
  routers/auth.py      # 엔드포인트
  deps.py              # get_current_user 의존성
alembic/               # 마이그레이션 (0001_initial)
tests/                 # pytest

API

메서드경로인증설명
POST/auth/signup-회원가입
POST/auth/login-access/refresh 토큰 발급
POST/auth/refresh-refresh 토큰 회전(rotation)
POST/auth/logoutBearerrefresh 토큰 무효화
GET/auth/meBearer내 정보
DELETE/auth/meBearer탈퇴 (소프트 삭제)

데이터 모델

  • users: id, email(unique), username, hashed_password, is_active, created_at
  • refresh_tokens: id, user_id(FK), token_hash, expires_at, revoked, created_at

4 UE5 클라이언트 ue5/AuthClient/

어느 프로젝트든 Plugins/에 넣으면 되는 독립 플러그인. 서버 통신은 서브시스템 하나가 전담하고, 화면은 로그인 · 회원가입 · 프로필 3개 위젯으로 분리했다.

구성

  • UAuthSubsystem (GameInstanceSubsystem)
    • HTTP 모듈로 6개 API 호출: Signup / Login / Refresh / Logout / Withdraw / GetMe
    • JSON 파싱, BlueprintAssignable 결과 델리게이트 (bool bSuccess, FString Message)
    • refresh 토큰을 SaveGame으로 영속화 → 재시작 시 자동 세션 갱신
  • UAuthScreenWidget — 화면 위젯 공통 베이스. 서브시스템 접근(GetAuth)과 화면 전환(SwapTo) 제공
  • UAuthLoginWidget — 로그인 화면. EmailBox / PasswordBox / LoginButton / SignupButton / StatusText
  • UAuthSignupWidget — 회원가입 화면. EmailBox / UsernameBox / PasswordBox / SignupButton / (BackButton) / StatusText
  • UAuthProfileWidget — 로그인 후 내 정보 화면. InfoText(생성 시 GetMe 자동 조회) / LogoutButton
  • UAuthSaveGame — refresh 토큰 저장용

화면 흐름

전환은 C++ 베이스의 SwapTo(다음 위젯)가 담당한다. 각 WBP는 디테일 패널 Auth|Navigation에서 이동할 위젯 클래스를 지정한다.

WBP_Login 회원가입→ WBP_Signup 가입성공 · 뒤로↩ WBP_Login
WBP_Login 로그인성공 · 자동로그인→ WBP_Profile 로그아웃↩ WBP_Login
로그인 회원가입 프로필

5 검증 상태

서버 로직 pytest 12개 통과 (인메모리 SQLite로 로직 검증).

실 MySQL 연동 MySQL 8.4.9에 auth_db 생성 + alembic upgrade head. signup→login→me→refresh(회전)→logout 전 흐름을 실 DB 대상으로 스모크 테스트 통과.

UE5 통합 (실동작) UE 5.7 프로젝트에서 플러그인 빌드 성공. PIE에서 회원가입→로그인 수행, DB로 확인: users에 bcrypt 해시 비밀번호 저장(평문 아님), refresh_tokens에 SHA-256 해시 토큰 저장(만료 14일).

주의 · Windows 서버를 cmd 콘솔로 띄울 때 창 안을 클릭/드래그하면 QuickEdit 모드로 프로세스가 멈춤. 멈추면 창 클릭 후 Esc/Enter. 로그는 눈으로만 볼 것.

6 참고 / 주의

  • bcrypt를 <4.1로 고정 — passlib 1.7.4가 bcrypt 4.1+와 호환되지 않는 알려진 이슈 때문.
  • JWT_SECRET, MySQL 접속 정보는 로컬 .env로 주입하며 커밋하지 않음(.gitignore 처리).
  • 이메일 인증·비밀번호 재설정·소셜 로그인은 학습 스코프 유지를 위해 제외.

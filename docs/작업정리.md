# 작업 정리 — Auth Server + UE5 Client

> FastAPI 기반 JWT 인증 백엔드와, 이를 호출하는 UE5 클라이언트 플러그인.
> 목적: 학습/실습 (FastAPI · JWT · MySQL · UE5 HTTP 연동)

작성일: 2026-07-15

---

## 1. 개요

언리얼 엔진 5에서 사용할 인증 시스템을 **서버 + 클라이언트** 두 파트로 구성했다.
두 파트는 하나의 REST API 계약(요청/응답 형식)을 공유한다.

| 파트 | 위치 | 상태 |
|------|------|------|
| FastAPI 인증 서버 | `server/` | ✅ 구현 + 테스트 12개 통과 + 실 MySQL 연동 검증 |
| UE5 클라이언트 플러그인 | `ue5/AuthClient/` (원본), `ue5/ViveCodingUE/Plugins/AuthClient/` (프로젝트 적용) | ✅ UE 5.7 빌드 성공 + PIE 실동작 검증 |
| UE5 테스트 프로젝트 | `ue5/ViveCodingUE/` | ✅ 플러그인 통합, WBP_Login + GameMode 구성 |

---

## 2. 핵심 결정 사항

- **언어/프레임워크**: Python + FastAPI
- **DB**: MySQL (PyMySQL 동기 드라이버), SQLAlchemy 2.0 ORM + Alembic 마이그레이션
- **인증**: JWT **Access + Refresh** 토큰
  - Access: 30분, 무상태 검증
  - Refresh: 14일, **DB에 SHA-256 해시로 저장** → 로그아웃·탈퇴 시 무효화 가능
- **탈퇴**: **소프트 삭제** (`is_active=false` + 모든 refresh 토큰 무효화)
- **UE5**: C++ HTTP 서비스(Subsystem) + 토큰 저장 + 로그인 UMG 위젯
- async 스택은 학습 복잡도 때문에 의도적으로 배제

## 3. 서버 (`server/`)

### 구조 (계층 분리)
```
app/
  main.py              # FastAPI 앱, 라우터 등록
  core/
    config.py          # .env 설정 로드
    database.py        # SQLAlchemy engine/session, get_db
    security.py        # 비밀번호 해싱(bcrypt), JWT 발급/검증, 토큰 해시
  models/              # User, RefreshToken ORM
  schemas/auth.py      # Pydantic 요청/응답
  services/auth_service.py  # 비즈니스 로직
  routers/auth.py      # 엔드포인트
  deps.py              # get_current_user 의존성
alembic/               # 마이그레이션 (0001_initial)
tests/                 # pytest
```

### API

| 메서드 | 경로 | 인증 | 설명 |
|--------|------|------|------|
| POST | `/auth/signup` | - | 회원가입 |
| POST | `/auth/login` | - | access/refresh 토큰 발급 |
| POST | `/auth/refresh` | - | refresh 토큰 회전(rotation) |
| POST | `/auth/logout` | Bearer | refresh 토큰 무효화 |
| GET | `/auth/me` | Bearer | 내 정보 |
| DELETE | `/auth/me` | Bearer | 탈퇴 (소프트 삭제) |

### 데이터 모델
- `users`: id, email(unique), username, hashed_password, is_active, created_at
- `refresh_tokens`: id, user_id(FK), token_hash, expires_at, revoked, created_at

### 실행 방법
```bash
cd server
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env          # 값 채우기 (DATABASE_URL, JWT_SECRET)
alembic upgrade head            # 테이블 생성
uvicorn app.main:app --reload   # http://127.0.0.1:8000/docs
```

### 테스트 결과
```
pytest -q  →  12 passed
```
검증 범위: 회원가입(성공/중복/검증오류), 로그인(성공/실패), 토큰 인증,
refresh 회전 + 재사용 차단, 로그아웃, 탈퇴 후 로그인·토큰 차단.

## 4. UE5 클라이언트 (`ue5/AuthClient/`)

어느 프로젝트든 `Plugins/`에 넣으면 되는 **독립 플러그인**.

### 구성
- **`UAuthSubsystem`** (GameInstanceSubsystem)
  - HTTP 모듈로 6개 API 호출: Signup / Login / Refresh / Logout / Withdraw / GetMe
  - JSON 파싱, `BlueprintAssignable` 결과 델리게이트 `(bool bSuccess, FString Message)`
  - refresh 토큰을 SaveGame으로 영속화 → 재시작 시 자동 세션 갱신
- **`UAuthLoginWidget`** (UserWidget 베이스)
  - `BindWidget`으로 에디터 위젯 자동 연결 (EmailBox / PasswordBox / UsernameBox / LoginButton / SignupButton / StatusText)
- **`UAuthSaveGame`**: refresh 토큰 저장용

### 사용
- C++: `GetGameInstance()->GetSubsystem<UAuthSubsystem>()`
- Blueprint: `Get Game Instance Subsystem (AuthSubsystem)` 노드
- 서버 주소가 다르면 `SetBaseUrl("http://<host>:<port>")`
- WBP_Login 제작 절차는 `ue5/AuthClient/README.md` 참고

## 5. 검증 상태

- ✅ **서버 로직**: pytest 12개 통과 (인메모리 SQLite로 로직 검증).
- ✅ **실 MySQL 연동**: MySQL 8.4.9에 `auth_db` 생성 + `alembic upgrade head`로 테이블 생성.
  서버 기동 후 signup→login→me→refresh(회전)→logout 전 흐름을 실 DB 대상으로 스모크 테스트 통과.
- ✅ **UE5 통합 (실동작)**: `AuthClient` 플러그인을 UE 5.7 프로젝트에 넣어 빌드 성공(`UnrealEditor-AuthClient.dll`).
  PIE에서 **회원가입 → 로그인**을 실제로 수행하고 결과를 DB로 확인:
  - `users`에 계정 저장, 비밀번호는 **bcrypt 해시**(`$2b$...`)로 저장(평문 아님)
  - 로그인 시 `refresh_tokens`에 **SHA-256 해시** 토큰 저장, 만료 14일, `revoked=false`

### 주의 (Windows)
- 서버를 `cmd` 콘솔로 띄울 때 **창 안을 클릭/드래그하면 QuickEdit 모드로 프로세스가 멈춤**.
  멈추면 창 클릭 후 `Esc`/`Enter`. 로그는 눈으로만 볼 것.

## 6. 참고 / 주의

- `bcrypt`를 `<4.1`로 고정 — passlib 1.7.4가 bcrypt 4.1+와 호환되지 않는 알려진 이슈 때문.
- `JWT_SECRET`, MySQL 접속 정보는 로컬 `.env`로 주입하며 커밋하지 않음(`.gitignore` 처리).
- 이메일 인증·비밀번호 재설정·소셜 로그인은 학습 스코프 유지를 위해 제외.
